Anthropic accidentally exposed Claude Code's entire TypeScript source code through a misconfigured npm package map file on the seventh day of Fable 5's suspension — handing competitors architectural intelligence at the worst possible competitive moment — as Intel surged 8.5% in premarket on a major Apple chip deal, OpenAI prepared a confidential IPO filing, and Anthropic opened a Seoul office.

The Claude Code source exposure is an operational security failure at the intersection of the two worst moments for Anthropic: Fable 5's seventh day offline has already disrupted enterprise AI coding momentum, and now the architectural details of Claude Code — its tool integration layer, context-window management, permission model, and hooks infrastructure — are publicly visible to competitors who had been reverse-engineering the product from the outside. The exposure is not a breach in the traditional sense: a misconfigured npm package map file made source code that should be private readable by anyone who knew to look. Whether competitors acted on the exposure before it was corrected, and for how long the misconfiguration was active, are questions Anthropic has not yet answered publicly. The competitive damage is asymmetric: the exposed code does not immediately enable a competitor to replicate Claude Code's capabilities, but it eliminates the obfuscation barrier that forces competitors to build from independent architectures. For enterprise security teams evaluating Claude Code contracts, the npm exposure raises a question about Anthropic's operational security practices that will need an explanation before procurement decisions are finalized. The Anchor

Anthropic's Claude Code source code was publicly visible via npm. For how long, and who saw it, the company has not said. It happened on day seven of Fable 5 being offline. The two incidents together define Anthropic's worst week in its enterprise AI coding market position.

A misconfigured npm package map file associated with Claude Code's TypeScript package made the product's full source code readable to anyone who accessed the npm registry with the right query. The code that was exposed includes the architectural decisions that make Claude Code a competitive product: the tool integration layer that connects the AI model to file systems, terminals, and external APIs; the permission model that governs what actions Claude Code can take autonomously versus what requires user approval; the context-window management that determines how much of a codebase Claude can hold in active consideration; and the hooks infrastructure that allows integrations with external services. These are the engineering decisions competitors spend months developing independently — the exposure hands them a map.

The npm exposure arrived on the same day Fable 5 entered its seventh consecutive day offline. The timing is coincidental but the compounding effect is real: enterprise procurement teams evaluating AI coding tools are now making decisions without access to Fable 5's superior benchmark performance (which was offline before most enterprise evaluations could test it) and with questions about Claude Code's operational security practices (surfaced by the npm exposure). The two incidents together have narrowed Anthropic's competitive window in the enterprise AI coding market to the period after Fable 5 is restored and after the npm exposure is formally explained to enterprise security teams — a window that does not yet have a date. Separately, Anthropic opened a Seoul office on Wednesday with announced partnerships across the Korean AI ecosystem, expanding into a market that has been a strong early adopter of enterprise AI tools.

A misconfigured npm package map file in Claude Code's TypeScript package made Anthropic's full Claude Code source code publicly accessible through the npm registry for an undisclosed period, according to reports Thursday. The exposed code includes Claude Code's tool integration layer, permission model, context-window management system, and hooks infrastructure — the engineering decisions that define the product's competitive position in the enterprise AI coding assistant market. Anthropic has corrected the misconfiguration but has not disclosed how long the package was accessible or whether any parties downloaded or analyzed the source before it was removed.

The competitive implications of the exposure are significant even if no competitor immediately acted on the code. The architectural decisions embedded in Claude Code's source — how it handles tool permissions, how it manages long-context coding sessions, what its hooks architecture looks like at the implementation level — are the product of months of engineering iteration that is not replicable from black-box product testing alone. Competitors who now have access to the full source can understand design tradeoffs that Anthropic made, identify potential limitations in the current architecture, and calibrate their own product roadmaps accordingly. For enterprise customers, the exposure raises a question about whether a company that misconfigured an npm package may have other operational security gaps — a question that will need formal explanation before security-sensitive enterprise procurement teams close contracts with Anthropic's products.

Quick hits

  • The npm exposure incident highlights a structural risk in the way AI coding tools are distributed. Claude Code is packaged and distributed through the npm ecosystem — the same package registry used by millions of JavaScript and TypeScript developers. The npm registry is a public-by-default distribution channel where package misconfiguration errors can expose private source code that is not intended for public access. Anthropic's incident is not the first time a company has accidentally exposed proprietary code through a misconfigured npm package, and it is unlikely to be the last — but the competitive sensitivity of Claude Code's architecture makes this incident more consequential than a typical open-source misconfiguration. The incident argues for a more careful review of how frontier AI company source code is packaged for distribution and whether separate private distribution channels should be used for proprietary tool implementations.
  • NovoCure fell 18.8% and Accenture fell 15.6% on Wednesday, two of the session's largest single-stock declines alongside the broad Warsh FOMC selloff. NovoCure's decline follows news that a major clinical trial for its tumor-treating fields (TTFields) technology in a new indication did not meet its primary endpoint — a company-specific negative that compounds the macro headwind. Accenture's decline follows a guidance revision that cited lower enterprise technology spending intent in the near term — a data point that is relevant for the broader enterprise AI sector, as Accenture's consulting revenues are a leading indicator for enterprise technology investment. If enterprise technology spending is softening, AI adoption timelines at large enterprises may be slower than the frontier AI companies' revenue growth implied.

The competitive landscape in June 2026 — Fable 5 offline, Cursor acquired by SpaceX-xAI, Claude Code source code exposed, OpenAI filing for IPO — gives Google an unusually clear competitive window to convert Gemini 3.5 Pro's internal testing into a launch that fills the market position Anthropic has vacated.
