# Morning Wire Codex Instructions

## Project Intent

Morning Wire is a static daily news site that is regenerated by a scheduled GitHub Actions workflow and deployed by Vercel. The core product quality bar is editorial trust: accurate source-backed summaries, concise synthesis, and no invented facts.

## Repository Map

- `public/index.html` is the daily hub.
- `public/tech-ai.html`, `public/crypto.html`, `public/markets.html`, `public/world.html`, and `public/science.html` are section deep dives.
- `public/styles.css` is the shared design system and should not be changed by the daily generation workflow.
- `.github/workflows/daily-build.yml` runs the scheduled build.
- `.github/prompts/daily-build.md` is the generation brief.
- `scripts/validate-edition.mjs` validates generated editions.
- `scripts/send-newsletter-email.mjs` sends the morning email when email is explicitly enabled.

## Working Rules

- Inspect the workflow, prompt, and current HTML before changing publishing behavior.
- Keep changes small and auditable; this repo has very little code, so hidden behavior is usually worse than explicit files.
- Do not send live email from local or CI unless `EMAIL_ENABLED=true` is explicitly set.
- Do not add package dependencies unless there is a clear operational need. Prefer Node 20 built-ins for scripts.
- Do not weaken validation to make a bad edition pass.
- Preserve the static-site design unless the user explicitly asks for visual changes.

## Editorial Rules

- Every claim in generated content must be traceable to a real linked source.
- Prefer primary sources and high-quality reporting over low-quality aggregators.
- Avoid filler, hype, vague "markets reacted" phrasing, and unsupported certainty.
- Prefer fewer, better stories over broad but thin coverage.
- If a topic is continuing coverage, label it as continuing coverage instead of pretending it is new.

## Verification

- Run `node scripts/validate-edition.mjs` after changing generated HTML, validation logic, or prompt rules.
- Run `node scripts/send-newsletter-email.mjs` only as a dry run unless the user explicitly wants a live send and the required secrets are set.
- For workflow changes, inspect `.github/workflows/daily-build.yml` for credential handling and live-send gating.
